## Decisions
- **Git: re-baseline from this Mac** (vs. recovering granular history from the other Mac). Code is
  identical via Syncthing; per-session history is narrated in `docs/sessions/`. The missing piece
  was a GitHub `origin`, now created — fixes the drift's root cause on this machine.
- **Multi-Mac protocol reaffirmed:** Syncthing moves files (`.git` excluded); git history must
  travel via `origin`. Any other Mac must `git remote add origin … && git fetch && git reset --hard
  origin/main` **before** committing, or history forks again. (Captured in memory
  `git-remote-reconciliation`.)
- **Feature-then-polish ordering:** Rename adds a 4th output-bar switch and TC disclosure adds a
  caret/panel to every queue row — both in the exact regions a polish pass touches. Build the
  feature first, polish the final layout once.
- **Rename state lives on `ConversionViewModel`, not `ConversionSettings`** — the pattern is
  session-only (spec: not persisted), and `ConversionSettings` is `Codable` and frozen onto every
  job in `queue.json`. The *resolved* output name already rides on `destinationURL`; only that needs
  freezing. (Deliberate deviation from the spec's "disposable" wiring note.)
- **Both features share one resolved recording-start `Date` per group** — `RecordingStartResolver`
  already returns date + `provenance`, so resolve once at enqueue and feed both rename `{date}/{time}`
  and the TC-disclosure origin tag.
