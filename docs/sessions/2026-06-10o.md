## 1. Codec ¬∑ resolution ¬∑ fps on recording rows (`6683656`, backlog 5b)

**Ask:** each recording row had empty horizontal space between the name and the SINGLE/SPLIT
badge; surface per-recording video params there (e.g. `HEVC ¬∑ 3840√ó2160 ¬∑ 25 fps`).

**Pure display add ‚Äî no new I/O.** The scan already reads `VideoStreamParams`
(`codecName`/`width`/`height`/`avgFrameRate`) per clip via ffprobe for the param guard, stored as
`SegmentStreamInfo?` on each `DJIClip`. The row reads `group.clips.first?.streamInfo?.video` ‚Äî
the first segment speaks for the whole group because the grouping gate (`DJIFolderReader`) refuses
to chain segments whose codec/res/fps don't match.

**Implementation:**
- Three pure formatters added to `CJFormat` (`DesignControls.swift`):
  - `codec(_:)` ‚Äî `hevc`/`h265` ‚Üí `HEVC`, `h264`/`avc1` ‚Üí `H.264`, others uppercased verbatim.
  - `resolution(width:height:)` ‚Äî `3840√ó2160` (true √ó sign).
  - `fps(_:)` ‚Äî `25 fps` / `29.97 fps` (2 decimals when fractional); `nil` rate ‚Üí empty string.
- `RecordingRow.streamSummary` computed property joins the parts with ` ¬∑ `; `nil` when ffprobe
  couldn't read the stream (row just omits it).
- A `Text(streamSummary)` placed between the name VStack (which holds `maxWidth: .infinity`, so it
  keeps the flexible space) and the `CJBadge`. The codec text keeps its intrinsic width, so the
  **title truncates first** when the row is cramped ‚Äî safe at the 1220 pt min window.
- `Theme.txt3`, 11 pt, `.monospacedDigit()`, `.lineLimit(1)`.

**Tests:** new `CJFormatTests.swift` (+7) covering codec mapping/case-insensitivity/unknown,
resolution, and fps whole/fractional/nil. Added via xcodegen regen. **245/245.**

**Live-confirmed** by the user on the `2CULL` card: rows show e.g. `HEVC ¬∑ 3840√ó2160 ¬∑ 25 fps`
(and a 29.97 fps single). Also confirmed the **wrong-folder empty state** works ‚Äî selecting a
non-media folder (`2CULL-2025-08`) showed "No video segments found in 2CULL-2025-08."

## 2. Font cleanup (`6adb44a`)

**Ask (after eyeballing ¬ß1):** *"I see some serif-like font used for tc and folder names. What
about using monospaced SF Pro to show TC and for the rest just SF Pro, bolded where necessary?"*

**Diagnosis:** the "serif-like" font was **SF Mono** (`design: .monospaced`). Audited every
monospaced usage and classified each against the user's rule (TC stays mono, names/paths ‚Üí SF Pro):

| Location | Was | Now |
|---|---|---|
| Segment-sublist filenames (`RecordingsList.swift:218`) | SF Mono | **SF Pro** (kept `.monospacedDigit()` for the dur/size columns) |
| Queue-row disclosure **Output** path (`QueuePanel.swift:446`) | SF Mono | **SF Pro** (added `.monospacedDigit()` for date digits) |
| Source TC / Applied TC (`QueuePanel.swift:403/412`) | SF Mono | **unchanged** ‚Äî TC stays mono (user wants this) |
| Console log (`:513`), Rename token field | SF Mono | **unchanged** ‚Äî terminal/token alignment, not a folder name |

Already SF Pro and **not** touched (they weren't the culprit): the path wells (top source bar +
Output bar, `CJPathWell` ‚Äî SF Pro medium + monospacedDigit) and the queue-row title
(`job.displayName` ‚Äî SF Pro semibold).

**Note:** `design: .monospaced` and `.monospacedDigit()` are different ‚Äî the former swaps the whole
typeface to SF Mono (the serif-ish look), the latter only makes digits tabular while letters stay
proportional SF Pro. The fix is exactly: drop `design: .monospaced` on names/paths, keep
`.monospacedDigit()` for column alignment.

**Display-only ‚Äî no test change.** Build green; **user confirmed** the look, asked to merge.
