## Build session (same day) — Wave 3.2 SRT offset stitcher

- `feature/wave3-srt-stitcher` off `main`; authored directly (one file + tests, slow builds).
- **3.2 `SRTStitcher`** — splices per-segment `.SRT` sidecars into one continuous, globally-
  numbered telemetry track aligned to the joined video. Each segment's cues shift by the
  cumulative **decoded container duration** (ffprobe `-show_format` → `format=duration`) of
  preceding segments — *not* cue math (which drifts: the last cue rarely lands on the true end,
  and DiffTime rounding accumulates). The container duration is the same quantity the concat
  demuxer uses to lay segments end-to-end, so SRT and video stay locked at every seam.
- **Missing-sidecar handling:** a segment with no `.SRT` contributes no cues but still advances
  the offset by its duration → a missing middle (or first) sidecar never shifts later cues out
  of alignment.
- **Shape:** pure `stitch([Segment]) -> Document` core (durations + parsed docs in, one doc out,
  fully unit-testable); `FFmpegWrapper.probeDurationMilliseconds` + high-level
  `stitchSRT(segments: [(video, srt?)])` in an extension, reusing `StreamParameterGuard.GuardError`.
  `format.duration` is a seconds string → `round(×1000)` ms (ms granularity keeps seam error far
  below one frame; cumulative Int sum stays clean).
- **11 tests:** 3-segment continuity/sequencing (the backpressure), missing first/middle sidecar
  offset advance, verbatim payload + wall-clock preservation, single-segment no-op, empty input,
  `stitchToString` round-trip, and **skippable real-ffprobe integration** (probe a 2 s clip ≈
  2000 ms; two-clip end-to-end where clip 2's cue lands at clip 1's *decoded* duration, proving
  offset ≠ cue math; no-sidecar → `nil`).
- **81 tests pass** (0 failures; +11 over 3.1; integration tests actually ran via Homebrew
  ffmpeg/ffprobe). Merged `--no-ff` to `main` (`d60d30b`).
- **Process note:** built directly on `main` then moved work to the feature branch via
  `checkout -b` before committing — house rule is branch *first*; corrected mid-flow.
