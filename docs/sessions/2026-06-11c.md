of TC math. Full rationale in Decisions.

## Decisions

### TC override is TC-only, not Date-based
The existing `ConversionSettings.dateOverride: Date?` controls `creation_time` AND the TC
stamp together (by resolving a Date through the whole chain). A manual TC override should
change **only** the `-timecode` ffmpeg argument, not the `creation_time` atom (which should
still use the best available date signal). Implemented by checking `timecodeStringOverride`
in `resolveJoinMetadata()` after date resolution, before `TimecodeFormatter.wallClockTimecode()`.

### Override lives on `ConversionJob`, not `ConversionSettings`
`ConversionSettings` is `let` on `ConversionJob`; adding a `var` to the job directly is
simpler. Excluded from `ConversionJob.CodingKeys` → session-only for free (not persisted to
`queue.json`). User confirmed: session-only is correct.

### `TimecodeField` init: `components` binding, not `timecode` binding
Use `init(components: Binding<Timecode.Components>, at: TimecodeFrameRate)` with plain
`@State var tcComponents: Timecode.Components = .zero` — avoids the `@TimecodeState`
requirement (which only applies to the `Binding<Timecode>` init).

### fps mapping: `Double` → `TimecodeFrameRate` via `stringValue`
TimecodeKit has no `init(_ double: Double)`. Map via
`TimecodeFrameRate(stringValue: String(format: "%.3g", fps))`, which works for all common
DJI rates (25, 29.97, 30, 50, 60). Fallback `.fps25`.

### `TimecodeDisclosure.build()` updated to show override as Applied TC
Avoids a separate "override" UI label. When `tcOverride != nil`, `build()` uses it as
`appliedTimecode` with `provenance = .manualOverride`. `QueueRow .task(id:)` keyed on
`"\(job.id)-\(job.timecodeStringOverride ?? "")"` so the disclosure rebuilds reactively.

### UI location: inside `TimecodeDisclosurePanel`, below Applied TC
Already the natural home — the caret is already expanded when the user is inspecting TC
details. Field is hidden when `disclosure.timecodeEnabled == false` (TC preservation off).

### Commit 4 — TC override UI in `TimecodeDisclosurePanel`

**File:** `QueuePanel.swift` — `TimecodeDisclosurePanel` + a file-private fps helper

fps helper (bottom of QueuePanel.swift):

```swift
@State private var overrideActive: Bool = false

```

UI (inside body, gated on `disclosure?.timecodeEnabled == true`):
```swift
HStack {
    Text("Override TC").font(.system(size: 11)).foregroundStyle(Theme.txt3)
    TimecodeField(components: $tcComponents, at: tcRate)
        .timecodeValidationStyle(.orange)
        .timecodeFieldInputStyle(.continuousWithinComponent)
    Button("Set") { applyOverride() }
        .buttonStyle(.plain).foregroundStyle(Theme.accent)
        .keyboardShortcut(.defaultAction)
        .font(.system(size: 11, weight: .medium))
    if overrideActive {
        Button { clearOverride() } label: {
            Image(systemName: "xmark.circle.fill").foregroundStyle(Theme.txt3)
        }.buttonStyle(.plain)
    }
}
if overrideActive {
    Text("Overrides the resolved TC above")
        .font(.system(size: 10)).foregroundStyle(Theme.txt3.opacity(0.7))
}
```

Apply / clear:
```swift
private func applyOverride() {
    let s = String(format: "%02d:%02d:%02d:%02d",
                   tcComponents.hours, tcComponents.minutes,
                   tcComponents.seconds, tcComponents.frames)
    queue.updateTimecodeOverride(for: job.id, timecode: s)
    overrideActive = true
}
private func clearOverride() {
    queue.updateTimecodeOverride(for: job.id, timecode: nil)
    tcComponents = .zero
    overrideActive = false
}
```

`.onAppear` + `.onChange(of: job.timecodeStringOverride)`:
```swift
private func loadOverrideFromJob() {
    if let s = job.timecodeStringOverride,
       let tc = try? Timecode(.string(s), at: tcRate, by: .allowingInvalid) {
        tcComponents = tc.components
        overrideActive = true
    } else {
        tcComponents = .zero
        overrideActive = false
    }
}
```

## Edge cases handled by the plan

| Case | Resolution |
|---|---|
| TC preservation off | Override field hidden (`timecodeEnabled == false`) |
| Frames > fps max | TimecodeKit `enforceValid` policy rejects in the field |
| App restart | Override not in `CodingKeys` → lost on restart (session-only, by design) |
| Unknown fps | `tcFrameRate()` falls back to `.fps25` |
| SRT stitching | Unaffected — uses Date resolution chain, not the TC override |
