# 2026-06-12 (b) ‚Äî Sortable recordings columns (backlog 12, ‚Üí v1.0.1)

**Focus:** Finder-style clickable column headers for the discovered-recordings list. First post-1.0
feature.

## Decision: UI style
- User chose **clickable column headers** over a "Sort by ‚ñæ" menu. Sort keys: **Name, Date,
  Duration, Size** (Files left as a static label ‚Äî not a chosen key).
- Confirmed this ships as **v1.0.1** (the app is already a notarized DMG; this is post-release).

## Implementation
**`ConversionViewModel.swift`**
- New `SortKey { found, name, date, duration, size }` + `@Published sortKey` / `sortAscending`.
- `setSort(_:)` ‚Äî Finder semantics: re-click active column flips direction; switching columns adopts
  that column's `defaultAscending` (Name A‚ÜíZ; Date/Duration/Size lead descending = newest/longest/
  biggest first). `resetSort()` on rescan.
- Sorting folded into the existing `filteredGroups` chokepoint (filter ‚Üí sort), **decorate-sort-
  undecorate** so each date resolves once, not per comparison. `groups`, selection, rename cache
  untouched ‚Äî only the view feed reorders.
- Comparator extracted as a **pure static `orders(_:before:by:)` over a plain `SortField`** value ‚Äî
  that's what made it unit-testable without a scanned card. Stable `groupIndex` tie-break so equal
  rows never jiggle; `localizedStandardCompare` for numeric-aware name order.

**`Views/RecordingsList.swift`**
- New `ColumnHeaderBar` + `SortLabel`; shared `CJRowMetrics` widths so the header's FILES¬∑DURATION¬∑
  SIZE pixel-align over the rows (NAME¬∑DATE share the left region ‚Äî date is a row sub-line, not its
  own column). Active column shows an up/down accent chevron.

**Tests:** +11 in `ConjoynTests/RecordingSortTests.swift` (direction state machine, numeric name
order, tie-break stability, nil-date policy). **Full suite 330 pass, 0 failures** (was 319-ish).

## Two bugs caught live (eyeball iteration)
1. **Header ballooned vertically** ‚Äî used `Color.clear.frame(width:)` for leading slots; `Color.clear`
   is **greedy in any unconstrained axis**, so it stretched the bar to fill all height and centered
   the labels in a tall band. Fix: padding-based leading inset (`CJRowMetrics.leadingInset`), no
   greedy spacers ‚Üí compact bar.
2. **"DURATION" wrapped to two lines** ‚Äî label wider than its 66 pt column. Widened `durationCol`
   66‚Üí80 (shared by header + rows so they stay aligned) + `.lineLimit(1)` guard.

## Bug caught live: resort beachball
- Symptom: re-sorting hung ~1s while thumbnails were still loading. User blamed thumbnails.
- **Real cause:** the date sort resolved recording-start dates **synchronously inside
  `filteredGroups`** ‚Äî which runs in view-body on the main thread ‚Äî opening up to 74 `.SRT` files
  **off the SD card**. Thumbnails (an `actor` + async FFmpeg) were never the blocker; the cold cache
  just coincided with thumbnail loading.
- **Fix (two parts):** (a) `sortField` now reads only the *already-resolved* cache, falling back to
  the in-memory embedded date ‚Äî a sort is pure memory, can't beachball. (b) `prewarmStartDates()`
  resolves all dates **off-thread** (`Task.detached`, `.utility`) right after scan, passing only
  `Sendable` values (`[(UUID, DJIClip?)]` + `Date?`), then bumps `@Published startDateRevision` so
  the list re-sorts into corrected order once warm. Synchronous `resolvedStartDate(for:)` kept for
  the rename path (user-triggered, selected-only). User confirmed "works fantastic."

## Decisions
- nil-date sort policy: **`nil ‚Üí .distantPast`** (undated rows cluster at one end, flip with
  direction). Flagged the Finder "always-last" alternative as a `TODO(you):` in `orders(...)` ‚Äî
  left at the default for now.
