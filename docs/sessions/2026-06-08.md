## Decisions
- **Store `DJIClip` duration as `Int64` value + `Int32` timescale backing, computed `CMTime`** —
  preserves exact CMTime (spec) while staying trivially Codable/Sendable; mirror of the URL→String
  idiom. CMTime built only at the boundary.
- **Make `StreamParameterGuard` param structs `Codable, Sendable` and embed on `DJIClip`** —
  no duplicate codec/res/fps fields; grouping (2.4) and the join guard read identical data.
- **`ConversionSettings` kept lean** (user pick): `outputDirectory`, `outputFilename`,
  `useFolderNameAsFilename`, `outputContainer{.mp4,.mov}`, `preserveTimecode`, `fixCreationDate`,
  `stitchSRT`, `reEncodeOnMismatch=false`, `deleteOriginalsAfterVerify=false`. Dropped P2's
  `processingMode`/`audioMapping`/`generateReport`/`includeChecksum`. Add knobs as features land.
- **One `ConversionJob` = one record-group** (user pick), not a whole folder — matches the
  concat-demuxer join (one group → one output) and the watch-folder "join when the group is
  complete" state machine. `folderName`/`sourceFolderURL` identify the source DCIM folder.
- **Free to rename P2 fields** — no shipped `queue.json` to keep compatible, so `cardName`→
  `folderName` etc. is safe (removes P2-isms before the 1.7 QueueManager port).
