### Issues found
From the screenshots (2 jobs: 1 single-file 0004_D, 1 split-4 0009–0012):

1. **"Joining…" label wrong for single-file jobs** — single clips aren't joined, they're copied.
   Should say "Processing…" for `clips.count == 1`.

2. **⚠ Duration chip on every completed job** — `verificationSection` filter is `severity > .pass`,
   which catches `.info` (normal ±1-frame genpts delta). Fix: `severity >= .warning`.

3. **mapStatus doesn't promote thorough hash pass to `.verified`** — when Tier 1 warns but Tier 2
   byte-exact hash passes, the seal should be green. Currently stays orange because `SourceTargetResult.passed`
   is worst-wins and the Tier 1 duration warning drags it below the threshold.

4. **Seal animation may not stop cleanly** — `spin = false` in `onChange` uses spring animation
   (`.default`) to snap the rotation back to 0°, which can look like continued spinning. Fix: use
   `withAnimation(nil) { spin = false }` for an immediate stop.

5. **ETA doesn't include verification time** — the per-row ETA hits `~0:00` then verification
   adds 30–60 s of untracked wall-clock (Tier 2 hashing 13 GB at ~340 MB/s). Deferred to a separate
   plan (needs output-file size + read-speed baseline).

6. **Queue restore is silent** — `queue.json` reloads with no UI signal; confusing to users.
   Fix: show a dismissable "Restored N jobs from last session" banner.

7. **Queue row loses recording info** — SINGLE/SPLIT badge, duration, file size visible in the
   recordings list disappear once jobs are queued. Fill the 140 pt empty metrics column.
