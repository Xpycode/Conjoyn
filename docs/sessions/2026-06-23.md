**❌ False positive (verified, discarded):** Agent A's "FSEvents rescans queue up → redundant ffprobe."
Wrong — `isRescanning` is set **synchronously before the first `await`**, so concurrent FSEvents tasks reach
their guard while task 1 is suspended at `await discover`, see `true`, and drop. The real concern is the
**hung** variant (#1), not redundant work.

**✅ Confirmed solid:** Swift 6 isolation (MainActor-isolated state, `weak self` hops, honest
`@unchecked Sendable`) · the **re-join-forever fix** (ledger loads at init, persists on insert, pinned by
`testLedgeredGroupIsNotReEnqueuedAfterRelaunch`) · FSEvents lifecycle order + idempotent start/stop +
`deinit→stop` · Codable fallback-to-defaults · no unguarded force-unwraps.
