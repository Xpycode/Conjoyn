## Findings (the diff)
- **Scan button rendered icon-only** (just the `viewfinder` glyph, reads like a fullscreen control)
  — a side effect of the 2026-06-10g native-toolbar migration: SwiftUI toolbars default `Label` to
  `.iconOnly`. The old custom `TitleBar` used a plain `Text`, so it never hit this. Fixed with one
  modifier. The app's *primary action* now reads "⊹ Scan" like the prototype.
- Everything else in Empty + Loaded matches within tolerance. **Caveat:** only 2 of 5 states were
  live-verified — Scanning (spinner), Running, Done (green footer, progress fills, status colors)
  are code-correct vs spec but not eyeballed this session.
