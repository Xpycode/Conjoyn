### 3. NEW BUG found at session close — minimum window size mangles the Output bar
Root view is `.frame(minWidth: 1000, minHeight: 640)` + `.windowResizability(.contentMinSize)`
(`ContentView.swift:49`, `ConjoynApp.swift`). At the 1000pt floor the **Output bar middle row breaks**:
the "Output" label clips to "utput" on the left, "Add to Queue" is cut off on the right, and
"Timecode from recording time" wraps to two lines (see screenshots). The bar's intrinsic width
exceeds 1000, but its elements truncate/wrap instead of forcing the window min, so `.contentMinSize`
never picks up the true minimum and SwiftUI center-clips the overflow.

**Reference screenshots saved:** `03_Screenshots/min-window-size_2026-06-10m/`
- `1_too-small-output-bar-mangled.png` — the broken state (≈ current 1000pt min).
- `2_comfortable-minimum.png` — width that feels comfortable.
- `3_choose-button-stops-moving.png` — narrowest width where the Output *Choose…* button stops
  shifting (i.e. the bar has reached its compressed layout) → this is the real min target.
