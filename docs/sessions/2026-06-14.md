## Session: 2026-06-14 (late eve) — FeedbackKit didn't actually work: fixed 3 layered bugs, proven end-to-end

### Goal
Status check → user asked "is the feedback feature properly wired?" → became a bug hunt. The eve
wire-up was structurally correct but **0.1.0 did not work**. Fixed it end-to-end and bumped Conjoyn.

### Progress
- **Verified wiring (all correct):** `project.yml` dep, `FeedbackCommands` in `.commands`, config
  (`appID:"conjoyn"`, `Theme.acc2`, `DiagnosticLogger.recentTail` bridge). Fixed one **stale comment**
  (`ConjoynApp.swift` said feedback "stays inert until conjoyn is allow-listed" — already deployed) (`b5daa51`).
- **Bug 1 — sheet never presented (FeedbackKit `0.1.1`).** `FeedbackCommands` attached `.sheet` to a
  hidden `.background` anchor on the *menu button*; menu items from a `Commands` builder aren't in any
  window's view hierarchy → no window for the sheet → click did nothing. **Fix:** present imperatively via
  a self-managed `FeedbackPresenter` (`NSHostingController`+`NSWindow`, document-modal sheet on the key
  window, standalone window fallback for agent apps). `FeedbackView` gained injectable `onClose` (falls
  back to `\.dismiss` for direct-`.sheet` callers). Self-retaining window pattern (nil on close/endSheet).
- **Bug 2 — Send looked dead (FeedbackKit `0.1.2`).** Send stays disabled until the **consent checkbox**
  is ticked (server-enforced); the blocking reason rendered in tiny grey caption = easy to miss. **Fix:**
  when consent is the *sole* remaining blocker (`consentIsOnlyBlocker` = speculative validate of a draft
  copy with `consent=true`), highlight the consent row + the hint by Send in the app accent.
- **Bug 3 — server rejected with a log attached (FeedbackKit `0.1.3`).** Client validated raw `body` but
  SENT `composedBody` (description + diagnostic-log tail). Conjoyn's `recentTail(80)` ≈ **5235 chars** >
  the server's **5000** body cap → server rejected the whole post as "Description is required (max 5000
  chars)". **Fix:** `FeedbackDraft.composedBody` now self-caps to `maxBody`, trimming the **oldest** log
  lines (keeps most-recent tail + a `…(earlier log trimmed)` note); typed description never truncated.
  +1 pkg test → 10.
- **Conjoyn bumped:** dep 0.1.0→0.1.3; **app `1.0`/100 → `1.0.1`/101** (commits `c09331d`/`8c474f1`/
  `18fa704`). 341 app tests / 1 skip / 0 fail throughout; ad-hoc Debug build green each round.
- **PROVEN END-TO-END:** real in-app submit (bug "fttttj") → 302 → green "sent" → landed on the **public
  board** (`apps.lucesumbrarum.com/feedback.html`, one unified list w/ per-item type badges, **no separate
  "bugs section"**); email stripped to the private log (PII split). Confirmed via live `public.json` (1
  conjoyn entry).
- Updated `PROJECT_STATE.md` (`82cf9eb`) + memory `feedbackkit-in-app-feedback`.

### Decisions
