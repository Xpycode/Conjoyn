### ETA fix #1 — oscillation (root cause + fix)
- Real 60-job 4K eyeball: footer swung "~3 min → ~3h 43m (while *Verifying…*) → ~2 min". 
- **Cause:** `remainingQueueSeconds` derived a *live* throughput off the active job
  (`activeBytes × progress / elapsed`) for the pending estimate. `progress` covers only the fast ffmpeg
  join (internal-SSD write); the PM-2 **staged move + auto-verify** tail isn't progress-tracked, so the
  sample read ~10× too fast mid-join and **collapsed toward zero during verify** (progress frozen at 1.0,
  elapsed climbing) → pending term exploded to hours, then crashed on the next fast job.
- **Fix:** dropped the live-throughput override (`QueueManager+Processing.swift`). +1 regression test
  `testRemainingQueueSecondsPendingStableAcrossActiveJobPhase` (pending estimate must be phase-independent).

### ETA fix #2 — adaptive session pace (user's idea)
- Even after #1, the flat **steady-state history** (70 MB/s) felt wrong: ground truth from the diagnostic
  log showed the **cold first file took ~5.9 min** (1.7× realtime, ~7.5 MB/s — 10× slower than history),
  then the drive warmed and jobs ran 8–17× realtime; whole run ≈ **13 min** vs ~10 est (close in aggregate).
- User asked: "can't the ETA extrapolate from the first file's bytes/time to the remaining bytes?" → yes.
- **Implemented:** `QueueManager.sessionBytesDone / sessionSecondsDone` accumulate per *completed* job
  (full join→move→verify wall-clock), reset each batch in `processQueue()`, incremented at the success
  site next to `recordConversion`. `remainingQueueSeconds` pending throughput preference is now
  **session pace → history → default**. A cold run honestly raises the estimate and converges as it warms;
  measured only at job boundaries so it can't oscillate within a job. +1 test
  `testRemainingQueueSecondsUsesObservedSessionPaceForPending`.

### Console hang fix
- User: "opening the console hangs the app" — confirmed Conjoyn pegged at **99.3% CPU**. 
- **Cause:** the PM-2 console rewrite renders **all** lines (source-capped at 5 000) as one
  `.textSelection`-enabled `Text` from an `AttributedString`, re-laid-out on every streamed line.
- **Fix:** `ConsoleSection` now renders only the **last 300 lines** (`maxRenderedLines`); header shows
  "N lines · showing last 300"; **Copy All** still copies the full `consoleLog`. (`QueuePanel.swift`)

### Diagnostics done along the way (not bugs)
