### 4. Overlap policy authored ÔÇö `TODO(user)` retired
The `WatchFolderManager.rejectionReason(forAdding:existing:)` guard is the correctness gate for
multi-folder: each entry has an **isolated ledger**, so two roots at the same tree (or one nested in
the other) would each enqueue the same clips Ôćĺ a double join. `pathsOverlap` (symlink-resolved
component-prefix compare) detects that case.

**Decision (user):** *block on last-known path.* An existing entry whose volume is currently
**offline** (bookmark won't resolve) must STILL block an overlapping add ÔÇö fall back to its persisted
`rootPath` so a later re-mount can't silently resurrect an overlapping pair into a double join.
Implemented as `let existingURL = entry.resolvedRootURL ?? URL(fileURLWithPath: entry.rootPath)`
(~2 lines; `rootPath` was already stored on every entry for exactly this offline case). Removed the
`TODO(user)` marker and rewrote the doc comment to record the policy.

- **+1 regression test** `testRejectionReasonBlocksOfflineEntryViaLastKnownPath` ÔÇö fakes an offline
  volume with a garbage bookmark (`Data([0x00,0x01,0x02])` Ôćĺ `resolvedRootURL == nil`, asserted as a
  precondition), then proves same-path + nested-path reject and a disjoint path still allows.
