## Progress
- **Clarified the starting state first.** The top bar already *was* the titlebar — a custom 52 pt
  `TitleBar` `HStack` rendered under `.hiddenTitleBar`, with traffic lights overlaying a hand-tuned
  `Spacer().frame(width: 64)` inset and an `NSViewRepresentable` `WindowConfigurator`
  (`titlebarAppearsTransparent`/`titleVisibility=.hidden`/`isMovableByWindowBackground`). So the ask
  was really "make it a **native `.toolbar`**" (App Shell Standard) vs. the custom `HStack`. Traced
  Penumbra's `ContentView` — it uses native `.toolbar { ToolbarItemGroup(.navigation/.principal/
  .primaryAction) } + .toolbarRole(.editor)` and needs **no** NSWindow hacks. Picked layout with the
  user (AskUserQuestion): **path well centered (`.principal`), Scan trailing (`.primaryAction`)**,
  wordmark/tagline dropped.
- **Migrated `ContentView.swift`** (branch `feature/native-toolbar`): deleted the `TitleBar` struct
  and `WindowConfigurator`, added a native `.toolbar` (reusing `CJPathWell` + the `.cjStandard` Scan
  button) + `.toolbarRole(.editor)`. Net **−71/+41 lines** — the native toolbar provides the drag
  region and traffic-light layout the custom code hand-rolled. Build green.
- **Tahoe Liquid Glass "bubbles."** First native build showed a translucent **glass capsule behind
  every toolbar item** (path well, Choose…, Scan). Diagnosed: macOS 26 (this Mac is Darwin 25.5.0)
  auto-enrolls *native* toolbar items into Liquid Glass **for apps built against the 26 SDK**; the old
  custom `HStack` never hit that path, and Penumbra/CropBatch are flat because they're **pre-Tahoe-SDK
  builds** (the "siblings don't do it" tell = SDK skew). Our items carry their own backgrounds → ugly
  double layer.
- **Fixed via `UIDesignRequiresCompatibility`.** App-wide opt-out of Liquid Glass (correct here — the
  whole app is a bespoke dark theme that never wants glass). **Trap hit + recorded:**
  `INFOPLIST_KEY_UIDesignRequiresCompatibility: YES` **failed silently** (the `INFOPLIST_KEY_*`
  mechanism only injects an Apple allowlist; verified absent via PlistBuddy). **Working fix:** a
  minimal hand-managed base `Conjoyn/Info.plist` holding only that one key, `INFOPLIST_FILE` pointed
  at it, `GENERATE_INFOPLIST_FILE: YES` kept — Xcode seeds from the base and **merges** the generated/
  allowlisted keys on top (verified: built plist has `UIDesignRequiresCompatibility=true` **and**
  `CFBundleDisplayName=conjoyn` **and** `CFBundleIconName=AppIcon`). User confirmed bubbles gone, flat
  toolbar matching the siblings.
- **Cleanup:** removed the now-orphaned `Theme.titlebar` color token; refreshed stale "the source bar
  IS the titlebar" comments in `ContentView`/`ConjoynApp`.
- **Tests:** ran the full suite after the Info.plist/Theme changes (they touch the test host) →

## Decisions / notes
- **Native toolbar over custom HStack** — adopts the App Shell Standard, deletes fragile hand-tuned
  chrome (the 64 pt traffic-light spacer, the NSWindow configurator). Trade-off accepted: a native
  toolbar can't pixel-match the design handoff's flat strip, and dragging an empty *content* area no
  longer moves the window (native = drag-from-toolbar). User saw and approved.
- **App-wide Liquid Glass opt-out** (`UIDesignRequiresCompatibility`) rather than a per-item fix —
  there's no clean public per-item opt-out, and the bespoke theme wants no glass anywhere.
